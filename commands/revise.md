# Revise Proposal Based on Review Feedback

Please revise the proposal document at `{{file}}` based on the architecture and security review notes found at the bottom of the document.

## Process

1. **Read the proposal**: Carefully read the entire proposal document, paying special attention to any review feedback sections at the bottom
2. **Identify issues**: Extract all architecture compliance issues and security concerns from the review notes
3. **Revise the proposal**: Update the proposal to address each issue raised by the reviewers:
   - Fix architectural compliance violations
   - Address security vulnerabilities and concerns
   - Incorporate recommended best practices
   - Update technical approaches as needed
4. **Maintain review history**: Keep the original review notes at the bottom of the document under a "## Review History" section
5. **Add revision notes**: Add a new section documenting what was changed and why

## Output

The revised proposal should:
- Address all critical issues from the reviews
- Maintain the original intent and goals of the proposal
- Include a changelog section showing what was revised
- Preserve the review feedback for historical reference

After completing the revision, provide a brief summary of the key changes made.
